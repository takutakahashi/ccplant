# ccplant 機能ガイド

ccplant の主要機能について詳しく解説するドキュメント集です。各機能の使い方、設定方法、ベストプラクティスを学ぶことができます。

## 目次

### コア機能

#### 1. [セッション管理](./sessions.md)
AI エージェントとの対話セッションの作成・管理機能です。

**主な機能:**
- 新規セッションの作成
- セッションのライフサイクル管理
- セッション設定のカスタマイズ
- セッションの閲覧とアクセス
- セッションの削除
- ソートとフィルタリング
- Webhook によるセッション再利用

**対象ユーザー:** すべてのユーザー

---

#### 2. [チャットインターフェース](./chat.md)
AI エージェントとリアルタイムで対話するためのチャット機能です。

**主な機能:**
- メッセージの送信
- ストリーミングレスポンス
- メッセージ履歴の表示
- キーボードショートカット (Cmd/Ctrl+Enter)
- フォントカスタマイズ (サイズ 12-20px、フォントファミリー)
- URL の自動検出とリンク化
- コードブロックの表示

**対象ユーザー:** すべてのユーザー

---

#### 3. [Webhook 統合](./webhooks.md)
GitHub などの外部イベントをトリガーにセッションを自動起動する機能です。

**主な機能:**
- GitHub Webhook 統合
- Webhook の作成と設定
- Go Template によるマッチング
- トリガー条件の設定 (path, method, go_template)
- セッション再利用
- Webhook 管理 (一覧、編集、削除)
- エラーハンドリング

**対象ユーザー:** 開発者、DevOps エンジニア

---

#### 4. [スケジュール実行](./schedules.md)
Cron 式を使用した定期的なセッション実行機能です。

**主な機能:**
- Cron ベースのスケジューリング
- スケジュールの作成と設定
- スケジュール管理 (有効化/無効化、編集、削除)
- 次回実行時刻の表示
- Leader Election によるワーカー管理

**対象ユーザー:** 開発者、DevOps エンジニア

---

#### 5. [エージェント管理](./agents.md)
利用可能な AI エージェントの管理とカスタマイズ機能です。

**主な機能:**
- エージェント設定
- エージェントマーケットプレイス/プラグイン
- エージェントのインストールと管理
- エージェントテンプレート

**対象ユーザー:** すべてのユーザー

---

### セキュリティとアクセス制御

#### 6. [認証](./authentication.md)
ユーザー認証とアクセス管理機能です。

**主な機能:**
- ログイン方法 (API キー、GitHub OAuth)
- AUTH_MODE 設定 (both, api_key, oauth_only)
- ユーザー向け GitHub OAuth セットアップ
- API キー管理
- Cookie ベース認証
- セッション永続化

**対象ユーザー:** すべてのユーザー、管理者

---

#### 7. [権限管理](./permissions.md)
チームベースの権限とアクセス制御機能です。

**主な機能:**
- RBAC システムの概要
- ユーザーロール (user, admin など)
- デフォルト権限 (session:create, session:list, session:delete, session:access)
- チームマッピング
- GitHub チーム統合
- 権限管理

**対象ユーザー:** 管理者

---

### 高度な機能

#### 8. [MCP サーバー](./mcp-servers.md)
Model Context Protocol サーバーの設定と管理機能です。

**主な機能:**
- MCP サーバーとは
- MCP サーバーの設定
- 利用可能な MCP サーバー
- セッション Pod 内の MCP サーバー

**対象ユーザー:** 開発者

---

#### 9. [プッシュ通知](./notifications.md)
ブラウザプッシュ通知機能です。

**主な機能:**
- ブラウザ通知の有効化
- 通知タイプ (セッション完了、エラー、スケジュール実行)
- VAPID 設定
- 通知設定

**対象ユーザー:** すべてのユーザー

---

#### 10. [設定管理](./settings.md)
グローバル設定と個人設定の管理機能です。

**主な機能:**
- グローバル設定 (API 設定、OAuth)
- 個人設定 (フォント、テーマ、通知)
- プロフィール管理
- Single Profile Mode

**対象ユーザー:** すべてのユーザー、管理者

---

## よくある使用例

### 1. コードレビューの自動化
**使用機能:** Webhook + エージェント

プルリクエストが作成されたときに自動的にコードレビューを実行します。

→ 詳細: [Webhook 統合](./webhooks.md#github-プルリクエストの自動レビュー)

### 2. 定期的なセキュリティスキャン
**使用機能:** スケジュール + エージェント

毎日特定の時刻にセキュリティスキャンを実行します。

→ 詳細: [スケジュール実行](./schedules.md#定期的なセキュリティスキャン)

### 3. チーム開発環境のセットアップ
**使用機能:** セッション管理 + 権限管理

チームメンバーが共有できる開発セッションを作成します。

→ 詳細: [権限管理](./permissions.md#チーム共有セッション)

### 4. カスタム AI アシスタント
**使用機能:** エージェント + MCP サーバー

特定のプロジェクトに最適化された AI アシスタントを構築します。

→ 詳細: [MCP サーバー](./mcp-servers.md#カスタムエージェントの構築)
